# Retningslinjer for dette prosjektet

Disse instruksene gjelder for hele repoet.

1. Kode skal skrives i Python og følge PEP 8 samt fungere med Black (linjelengde 88).
2. Bruk type hints der det er naturlig, og skriv kode som passerer Ruff uten ekstra advarsler.
3. Forklar endringer enkelt og forståelig i commit- og PR-meldinger på norsk.
4. Foreslå enkle pytest-tester for viktige funksjoner når du legger til ny logikk.
5. Unngå overdrevent kompliserte løsninger dersom en enklere variant fungerer like bra.
